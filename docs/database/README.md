## Naming Conventions

### Tables

| Rule | Example |
|------|---------|
| **snake_case** | `article_tags`, `dining_offers` |
| **Singular nouns** for entities | `user` not `users` (TypeORM entity) -- but table names are **plural** by convention in this project |
| **Join tables** combine both table names | `restaurant_cuisines`, `article_tags` |
| **Prefix domain** for sub-tables | `classified_messages`, `product_variants` |

> **Note on TypeORM mapping:** Entity class names are PascalCase singular (`User`, `Article`). The `@Entity('users')` decorator maps to the plural snake_case table name. All entities extend a `BaseEntity` or use a shared column set for timestamps.

### Columns

| Rule | Example |
|------|---------|
| **snake_case** | `created_at`, `user_id`, `price_range` |
| **Foreign keys** use `<referenced_table_singular>_id` | `user_id`, `category_id`, `venue_id` |
| **Boolean columns** use `is_` prefix | `is_recurring`, `is_free`, `is_active` |
| **Timestamps** use `_at` suffix | `created_at`, `published_at`, `deleted_at` |
| **Enum columns** stored as PostgreSQL `enum` types | `role`, `status`, `price_range` |
| **JSONB columns** documented with expected shape | `social_links`, `opening_hours`, `attributes` |

## Primary Keys

All tables use **UUID v4** primary keys generated by PostgreSQL:

```sql
id UUID PRIMARY KEY DEFAULT gen_random_uuid()
```

**Rationale:**
- No sequential ID enumeration (security)
- Safe for distributed systems and future microservice splits
- Mergeable across environments (dev/staging/prod)
- TypeORM maps via `@PrimaryGeneratedColumn('uuid')`

**Trade-off:** UUIDs are 16 bytes vs 4 bytes for `int`. B-tree index pages hold fewer entries. For this application scale (city guide, not billions of rows), the overhead is negligible.

## Timestamp Columns

Every table includes:

```sql
created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
```

TypeORM decorators:

```typescript
@CreateDateColumn({ type: 'timestamptz' })
created_at: Date;

@UpdateDateColumn({ type: 'timestamptz' })
updated_at: Date;
```

All timestamps are stored in **UTC** (`TIMESTAMPTZ`). The application layer converts to `Europe/Berlin` for display.

## Soft Delete Pattern

Content tables that support soft deletion include:

```sql
deleted_at TIMESTAMPTZ DEFAULT NULL
```

When `deleted_at IS NOT NULL`, the record is considered deleted. TypeORM provides `@DeleteDateColumn()` and the `softDelete()` / `softRemove()` methods.

**Tables using soft delete:**
- `users`
- `articles`
- `guides`
- `events`
- `restaurants`
- `videos`
- `competitions`
- `classifieds`
- `products`
- `media`

**Partial indexes** exclude soft-deleted rows to keep queries efficient:

```sql
CREATE INDEX idx_articles_published
  ON articles (published_at DESC)
  WHERE deleted_at IS NULL AND status = 'published';
```

**Hard delete** is reserved for:
- `refresh_tokens` (expired tokens are purged by cron)
- `cart_items` (removed from cart)
- `classified_messages` (GDPR erasure)
- Join tables (`article_tags`, `restaurant_cuisines`, `video_tags`)

## TypeORM Entity Mapping

### Base Entity Pattern

```typescript
// src/common/entities/base.entity.ts
import {
  PrimaryGeneratedColumn,
  CreateDateColumn,
  UpdateDateColumn,
  DeleteDateColumn,
} from 'typeorm';

export abstract class BaseEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @CreateDateColumn({ type: 'timestamptz' })
  created_at: Date;

  @UpdateDateColumn({ type: 'timestamptz' })
  updated_at: Date;
}

export abstract class SoftDeletableEntity extends BaseEntity {
  @DeleteDateColumn({ type: 'timestamptz', nullable: true })
  deleted_at: Date | null;
}
```

### Enum Registration

PostgreSQL enums are created in migrations and mapped in entities:

```typescript
// Migration
await queryRunner.query(`
  CREATE TYPE user_role AS ENUM ('user', 'editor', 'admin', 'super_admin');
`);

// Entity
@Column({ type: 'enum', enum: UserRole, default: UserRole.USER })
role: UserRole;
```

### Relationship Patterns

```typescript
// Many-to-One (article -> author)
@ManyToOne(() => User, { onDelete: 'SET NULL' })
@JoinColumn({ name: 'author_id' })
author: User;

// Many-to-Many via join table
@ManyToMany(() => Tag)
@JoinTable({
  name: 'article_tags',
  joinColumn: { name: 'article_id' },
  inverseJoinColumn: { name: 'tag_id' },
})
tags: Tag[];
```

## Database Extensions

Required PostgreSQL extensions:

```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";    -- UUID generation (fallback)
CREATE EXTENSION IF NOT EXISTS "pgcrypto";     -- gen_random_uuid() preferred
CREATE EXTENSION IF NOT EXISTS "pg_trgm";      -- Trigram similarity for search
CREATE EXTENSION IF NOT EXISTS "unaccent";     -- Accent-insensitive search (German umlauts)
```

## Connection Configuration

```typescript
// src/config/database.config.ts
import { TypeOrmModuleOptions } from '@nestjs/typeorm';

export const databaseConfig: TypeOrmModuleOptions = {
  type: 'postgres',
  host: process.env.DB_HOST || 'localhost',
  port: parseInt(process.env.DB_PORT, 10) || 5432,
  username: process.env.DB_USERNAME || 'iloveberlin',
  database: process.env.DB_DATABASE || 'iloveberlin',
  password: process.env.DB_PASSWORD,
  entities: ['dist/**/*.entity.js'],
  migrations: ['dist/migrations/*.js'],
  synchronize: false,  // NEVER true in production
  logging: process.env.NODE_ENV === 'development',
  ssl: process.env.NODE_ENV === 'production' ? { rejectUnauthorized: true } : false,
  extra: {
    max: 20,           // connection pool size
    idleTimeoutMillis: 30000,
  },
};
```
